# Flutter Weather App

> **This project was built to test Opus 4.7's coding capabilities.** The entire app — architecture, UI, animations, and CI/CD — was generated by Claude Code (Opus 4.7) with minimal human guidance.

A Xiaomi Weather-inspired Flutter application featuring an immersive edge-to-edge experience, dynamic per-weather animations, GPS-based location detection, and rich weather data visualization powered by the free Open-Meteo API.

## Features

- Immersive full-screen weather experience (edge-to-edge, transparent status bar, dynamic gradients)
- Dynamic weather animations implemented with pure Flutter CustomPainter:
  - Sun / moon halo with breathing glow
  - Drifting clouds
  - Falling rain particles (light & heavy)
  - Swaying snowflakes
  - Thunderstorm with periodic lightning flashes
  - Fog bands
- GPS-based automatic location detection with graceful permission handling
- Manual city search (debounced, multi-language) via Open-Meteo geocoding
- Current conditions: temperature, apparent temperature, humidity, wind, pressure, UV index
- 24-hour forecast (horizontal scroll) and 7-day forecast (with min/max range bars)
- Air Quality Index (AQI) card with color gradient & PM2.5 / PM10
- Animated sunrise / sunset arc reflecting current sun position
- Life indices: clothing, sports, car wash, UV protection
- Offline cache via `SharedPreferences` — last-known weather is shown when the network fails
- Pull-to-refresh

## Tech Stack

- **Flutter** — cross-platform (Android / iOS)
- **Riverpod** — state management
- **Dio** — HTTP client
- **Geolocator** — GPS positioning
- **SharedPreferences** — local cache
- **intl** — date/locale formatting (Chinese weekday names)
- **Open-Meteo API** — free weather/AQI/geocoding, no API key required
- **GitHub Actions** — automated Android APK builds on every `main` push

## Architecture

Clean Architecture layers under `lib/`:

```
lib/
  core/          constants, theme, formatters
  data/          API clients, cache, repository implementation
  domain/        entities, repository interface
  presentation/  providers, screens, widgets (incl. weather animation layers)
```

## Data Source

[Open-Meteo](https://open-meteo.com/) — free non-commercial weather API, no registration needed.

Endpoints in use:
- Forecast: `api.open-meteo.com/v1/forecast`
- Air Quality: `air-quality-api.open-meteo.com/v1/air-quality`
- Geocoding: `geocoding-api.open-meteo.com/v1/search`

## Building

Local build:

```bash
flutter pub get
flutter run
```

Android release APKs are built automatically by GitHub Actions — download them from the Actions tab artifacts, or from the GitHub Release attached to any `v*` tag. No local Android SDK required.

## Requirements

- Flutter 3.x (Dart SDK ^3.11.4)
- Android minSdk 21+
- Internet + Location permissions (declared in `AndroidManifest.xml`)
